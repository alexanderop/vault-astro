Comprehensive guide on using Claude CLI's print mode (`claude -p`) as a primitive for building agentic applications. Key insight: `claude -p` isn't just a CLI wrapper — it's a full-featured agent runtime.

## Key Concepts

**Input/Output Format Matrix** — Think of print mode as a state machine:

- `text → text`: Simple pipes, human-readable
- `text → json`: Structured responses with metadata
- `text → stream-json`: Real-time streaming UI (requires `--verbose`)
- `stream-json → stream-json`: Bidirectional streaming (full duplex)

**5 Ways to Feed Prompts** — CLI argument, stdin pipe, stdin redirect, here-doc, and stream-json input.

**Structured Output** — Use `--json-schema` for type-safe extraction. Response includes a `structured_output` field separate from the text `result`.

**Tool Configuration** — Disable all tools (`--tools ""`), whitelist specific tools, or use pattern-based allow/deny (`--allowedTools "Bash(git:*)"`)

**Permission Modes** — `--permission-mode bypassPermissions` for autonomous execution in sandboxed environments.

**Session Management** — Ephemeral (`--no-session-persistence`), fixed session ID for multi-turn, or `--continue` for most recent.

## Gotchas

- `--output-format stream-json` requires `--verbose`
- Stream input requires stream output (can't mix formats)
- MCP tools load regardless — use `--strict-mcp-config` to disable
- JSON schema output lives in `structured_output` field, not `result`
- Context caching reduces cost on subsequent runs
